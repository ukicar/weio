This document presents a rather small subsample of the original `AsciiDoc User Guide'' by _Stuart Rackham_ <srackham@methods.co.nz>, stressing the minimalism of AsciiDoc formatting. For compactness, each element is pesented only once. No example code is given, if possible. The reader is supposed to compare the source text file link:asciidoc-mini.txt[] and the result `asciidoc-mini.html document himself.

Formatting rules of AsciiDoc are very strict but simple. For example, the Header must start on the first line of the document and must begin with a document Title. Author and Revision lines immediately follow the title. The author line contains the author’s name optionally followed by the author’s email address. The email address comes last and must be enclosed in angle <> brackets.

The Preamble is an optional untitled section body between the document Header and the first Section title. It may consist of one or several Paragraphs each terminated by a blank line.

Sections

Different section levels are supported:

Level One Section

Level Two Section

Level Three Section

Level Four Section

The sections will not be numbered in the output asciidoc-mini.html document if you simply run

asciidoc asciidoc-mini.txt

To have sections with numbers, use option -a numbered

asciidoc -a numbered asciidoc-mini.txt

Note: Level Four Section titles are not numbered.

Two line titles

A two line title consists of a title line, starting hard against the left margin, and an underline which spans the width of the preceding title (plus-minus 1—3 characters).

One line titles

One line titles consist of a single line delimited by "=" characters:

= Document Title (level 0) =
== Section title (level 1) ==
=== Section title (level 2) ===
==== Section title (level 3)
===== Section title (level 4) =====

Note: The trailing delimiter is optional.

Admonition Paragraphs

NOTE:, TIP:, IMPORTANT:, WARNING: or CAUTION: at the start of a line have special meaning. They start admonition paragraphs or blocks. For example, Notes:

Note
 A one paragraph note

or

Note

A block note.

It can span multiple paragraphs and include other blocks, e.g., Lists.

References

Several types of references are available:

  • standard http:, https:, ftp:, file:, mailto: and callto: URLs in the form The AsciiDoc home page or http://www.methods.co.nz/asciidoc/

  • link to a local document like asciidoc-mini.txt or this

  • internal cross-reference, e.g. to Sections

Images

Images like or can be included anywhere in the text.

Tip

Standard icons for NOTE:, TIP:, IMPORTANT:, WARNING: or CAUTION: paragraphs and blocks are included automatically if the document is built with option -a icons:

asciidoc -a icons asciidoc-mini.txt

Put corresponding images to the ./images/icons/ directory or specify path to the icons with option -a iconsdir.

Table of Contents

Table of Contents is added automatically to the start of the document if option -a toc is used

asciidoc -a toc asciidoc-mini.txt

Final Remarks

Part of AsciiDoc syntax shown here presents the most common document formatting functions. Preference is given to those markup commands which look natural in the original ASCII text file. They can even improve rather than degrade its readablity.

For the complete list of AsciiDoc functions, see the official AsciiDoc User Guide.

This document is supposed to be built by the following command

asciidoc -a toc -a icons -a numbered asciidoc-mini.txt

with standard icons /etc/asciidoc/images/icons/* copied to ./images/icons/ under the current directory.
